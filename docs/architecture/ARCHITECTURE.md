# Lobster Journey 架构设计

> 分层架构：公司组织形式

---

## 架构概览

```
┌─────────────────────────────────────────────┐
│           战略层 (Strategy Layer)            │
│   运营目标、KPI、品牌定位、长期规划          │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│           战术层 (Tactics Layer)             │
│   内容策略、互动策略、增长策略               │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│           执行层 (Execution Layer)           │
│   自动化工具、机器人、定时任务               │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│             数据层 (Data Layer)              │
│   数据采集、分析、存储、可视化               │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│         基础设施层 (Infrastructure)          │
│   GitHub仓库、配置管理、监控告警             │
└─────────────────────────────────────────────┘
```

---

## 层级详解

### 1. 战略层 (Strategy Layer)

**职责**：定义方向和目标

**组件**：
- 品牌定位
- 运营目标（粉丝增长、内容质量）
- KPI体系
- 长期规划（100天、1年、3年）

**文档**：
- `docs/STRATEGY.md`
- `docs/BRAND_GUIDE.md`
- `docs/OKR.md`

---

### 2. 战术层 (Tactics Layer)

**职责**：制定具体策略

**组件**：
- 内容策略（选题、形式、发布时间）
- 互动策略（点赞、评论、收藏规则）
- 增长策略（如何获取流量）
- 风控策略（合规、红线）

**文档**：
- `docs/CONTENT_STRATEGY.md`
- `docs/INTERACTION_STRATEGY.md`
- `docs/GROWTH_STRATEGY.md`

---

### 3. 执行层 (Execution Layer)

**职责**：自动化执行

**组件**：
- 浏览器自动化（Playwright）
- 内容生成（AI写作、图片生成）
- 互动机器人（点赞、评论、收藏）
- 定时任务调度（Cron）

**代码**：
- `src/interaction/` - 互动模块
- `src/content/` - 内容生成模块
- `src/scheduler/` - 调度模块

---

### 4. 数据层 (Data Layer)

**职责**：数据驱动决策

**组件**：
- 数据采集（笔记数据、用户数据、互动数据）
- 数据分析（趋势分析、效果评估）
- 数据存储（本地数据库、GitHub）
- 数据可视化（报表、仪表盘）

**代码**：
- `src/data/collector.py`
- `src/data/analyzer.py`
- `src/data/storage.py`

---

### 5. 基础设施层 (Infrastructure Layer)

**职责**：提供底层支持

**组件**：
- GitHub仓库管理
- 配置管理（环境变量、密钥）
- 监控告警
- 日志系统

**代码**：
- `src/config/`
- `scripts/`
- `.github/workflows/`

---

## 技术栈

### 核心技术
- **Python 3.10+**：主要开发语言
- **Playwright**：浏览器自动化
- **Claude/GPT**：AI内容生成
- **SQLite**：本地数据存储
- **GitHub**：代码和文档管理

### 辅助工具
- **OpenClaw**：智能体框架
- **即梦**：图片/视频生成
- **小红书MCP**：小红书API

---

## 开发原则

### 1. 分层解耦
- 每层独立开发
- 通过接口通信
- 易于测试和维护

### 2. 配置驱动
- 所有策略可配置
- 不硬编码敏感信息
- 支持环境切换

### 3. 数据驱动
- 每个决策有数据支撑
- 自动记录运营数据
- 定期生成分析报告

### 4. 安全合规
- 严格遵守平台规则
- 控制操作频率
- 模拟人类行为
